### Pillar 2: ION Connect

**Tagline:** Decentralized social media platform, enhancing user control over content.

**Core Nature:** Social layer / Relationship layer / Content control layer / Community propagation layer

**Product Manifestations:**
- **Online+** — Tokenized Communities, turning audiences into economies
- **PUMPit** — Turn any X post into a token

**For ION DEX:**
- Community, content propagation, relationship graphs
- Future: trading + identity + social integration
- Channel / community / ecosystem communication alignment
- **Priority: P2 / Tier 3 — brand/narrative alignment first, product integration mid-term**

---

### Pillar 3: ION Liberty

**Tagline:** Powerful decentralized proxy and CDN, promoting digital freedom and privacy.

**Core Nature:** Access layer / Distribution layer / Anti-censorship layer / Network freedom layer / Privacy protection layer

**Product Manifestation:**
- **Frostbyte** — Bandwidth sharing product
  - Transform unused bandwidth into passive earnings
  - Part of ION Liberty
  - Has APK download (`apk.ice.io/frostbyte.apk`)
  - Multi-platform support
  - Already running with real users

**For ION DEX:**
- Front-end distribution, anti-censorship access, global reachability
- Infrastructure independence
- Future: truly decentralized front-end / content delivery
- **Priority: P2 / Tier 3 — strategic long-term, not first product-layer integration**

---

### Pillar 4: ION Vault

**Tagline:** Private, secure, decentralized storage with quantum-resistant encryption.

**Core Nature:** Storage layer / Data sovereignty layer / Long-term asset carrier / Privacy data home

**For ION DEX:**
- Future data assets need official-grade storage: identity data, merchant profiles, product info, fulfillment records, insurance credentials, logistics proofs, brand content assets, protocol & document assets
- Should not continue designing data layers with Web2 assumptions
- **Priority: P1 / Tier 2 — mid-term skeleton inclusion required**
