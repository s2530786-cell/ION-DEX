# 异常/维护公告模板

## A. 计划维护通知

```
ION DEX Maintenance Notice | {YYYY-MM-DD}

[中]
计划维护通知

时间：{开始时间} — {预计结束时间}（UTC+8）
范围：{受影响功能}
原因：{简要原因}

维护期间 {受影响功能} 将暂时不可用。资产始终安全，无需担心。

[EN]
Scheduled Maintenance Notice

Time: {Start} — {Estimated End} (UTC+8)
Scope: {Affected features}
Reason: {Brief reason}

{Affected features} will be temporarily unavailable during maintenance. All assets remain secure.
@iondex888
```

## B. 异常事件公告

```
ION DEX Incident Report | {YYYY-MM-DD}

[中]
我们注意到 {异常描述}。

当前状态
- 影响范围：{范围}
- 用户资产：{安全/受影响}
- 处理进度：{进度}

我们正在积极处理，将在 1 小时内更新进展。

[EN]
We are aware of {incident description}.

Current Status
- Impact scope: {Scope}
- User assets: {Safe / Affected}
- Progress: {Progress}

We are actively addressing this and will provide an update within 1 hour.
@iondex888
```

## C. 事件复盘报告

```
ION DEX Post-Incident Review | {YYYY-MM-DD}

[中]
事件复盘

事件：{事件描述}
时间：{发生时间} — {恢复时间}
根因：{根本原因}
修复：{修复措施}

预防措施
- {措施1}
- {措施2}

该事件已完全解决，所有资产安全。

[EN]
Post-Incident Review

Incident: {Description}
Duration: {Start} — {Resolution}
Root Cause: {Root cause}
Fix: {Applied fix}

Preventive Measures
- {Measure 1}
- {Measure 2}

This incident has been fully resolved. All assets are secure.
@iondex888
```

### 发布原则

- 计划维护：提前 24 小时通知
- 异常发现：15 分钟内发布初版公告
- 事件更新：每 1 小时更新一次，直到解决
- 复盘报告：解决后 24 小时内发布
- 诚实透明：不隐瞒、不缩小、不推卸
