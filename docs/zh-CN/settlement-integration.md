# 结算集成

> 面向商户结算、对账与验证链路的中文公开导读。

## 目标结算架构

```text
用户支付（USDT / USDC / ION）
  -> ION DEX 支付层
    -> 经由 DEX 流动性池完成换算或路由
    -> ION 进入商户结算钱包
    -> 平台费按目标逻辑拆分
    -> webhook 发出结算通知
    -> 浏览器验证链接生成
```

## 默认结算方向

- 默认以 **ION** 结算
- 部分场景可设计为自动转换到稳定币

## 费率说明

公开文档中的商户结算费率与拆分逻辑当前仍应视为**草案**：

- 公开常见平台费类：`0.5%`
- 销毁、质押、国库拆分仍需最终已审计合约和部署文档确认

## 对账方向

### 日结算报告

目标中支持：

- 支付总数
- 总金额
- 总费用
- 净结算金额
- 对应交易哈希与浏览器链接

### 手动对账

目标中支持按时间区间、币种、状态等条件查询 settlement 历史。

## 浏览器验证

目标中的结算记录应能映射到：

- 主支付交易
- 费用扣除记录
- 销毁记录
- 质押 / 国库分发记录

## 跨链结算

在目标设计中：

- ION 主网可直接结算
- BSC 及其它链可通过桥路径处理
- 不同链的确认时间会明显不同

## 风险与边界

- webhook 需要签名校验
- 大额退款 / 高风险结算路径不应无门禁执行
- 浏览器验证应服务于透明度，而不是替代风控
- 本页不能被解读为“所有结算能力已经生产可用”
