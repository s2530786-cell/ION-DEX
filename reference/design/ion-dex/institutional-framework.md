# 前端开发团队制度框架

## 4 份制度文件
1. 01-permission-matrix.md — 权限矩阵（18角色 × 5层 × R0-R3分级）
2. 02-acceptance-checklist.md — 验收清单（10节50+检查项）
3. 03-fix-escalation.md — 修复层级图（L0-L4分级 + 升级链路 + SLA）
4. 04-gate-rules.md — 升级阀门规则（P0→P1→P2→发布 四级阀门）

## 角色权限分级
- R0: 只读
- R1: 可提议 (suggest)
- R2: 可修改 (modify)
- R3: 可批准 (approve)

## 阀门条件
- P0: 11角色检查 + 4系统条件
- P1: 7角色检查
- P2: 4角色验证
- Release: 最终生产就绪检查
